### Tasks description for Lesson 1:

#### Задание 1. Диаграммы.

Рассмотрим вендинговый автомат продажи железнодорожных билетов.

Построим три диаграммы:

1. диаграмму вариантов использования (прецедентов) - *Usage variants diagram* - ***uvd.drawio***
2. диаграмму взаимодействия (последовательности) - *Sequence diagram* - ***sd.drawio***

Диаграмма нарисована для основного сценария работы - продажи билетов. При этом покупатель выбирает место назначения и отправления, затем поезд (варианты хранятся в базе данных Ж/Д), затем выбирает способ оплаты, оплачивает - и получает билет. Для возможности оплаты картой в терминал встроен модуль связи с банком. Также имеется три аппаратных модуля: считыватель карт, приемник банкнот и устройство печати билетов. Чтобы диаграмма не получилась чрезмерно широкой - я объединил их все в одну вертикальную группу, имея в виду, что это все - элементы аппаратной системы.

3. диаграмму классов - *Classes diagram* - ***cd.drawio***

#### Задание 2. Файловый менеджер.

За основу работы над файловым менеджером я думаю взять, прообраз заготовки из предыдущего нашего курса по введению в C#. Собственно, моя курсовая работа находится тут: https://github.com/JackenQuake/FileManager

Для этого файлового менеджера нет особенно интересных диаграмм использования и взаимодействий, поскольку все взаимодействия сводятся к "нажать кнопку" -> "получить реакцию". Но вот иерархия классов там была - компоненты для реализации простейшего оконного интерфейса в текстовом режиме.
Для иерархии классов - я тоже нарисовал диаграмму классов - *File Manager classes diagram* - ***FM_cd.drawio***
